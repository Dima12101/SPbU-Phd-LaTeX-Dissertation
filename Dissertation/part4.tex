\chapter{ГИБРИДНЫЙ МЕТОД (MA-VCG-QMIX)}\label{sec:ch4}

\section{Обоснование гибридного подхода}

Анализ существующих методов показал, что ни один из подходов не обеспечивает одновременно адаптивность, справедливость и устойчивость к неполной информации.

Аукционные механизмы (в частности, VCG) гарантируют стратегическую правдивость и максимизацию социального благополучия, однако не учитывают динамику среды. Методы многоагентного обучения с подкреплением (MARL), напротив, обеспечивают адаптивность, но не гарантируют справедливость распределения ресурсов и могут приводить к стратегически неоптимальному поведению агентов.

В связи с этим предлагается гибридный метод, объединяющий:
\begin{itemize}
	\item аукционный механизм распределения ресурсов (VCG);
	\item модель Dec-POMDP;
	\item алгоритм многоагентного обучения с подкреплением (QMIX).
\end{itemize}

\section{Концепция гибридного метода}

Предлагаемый метод основан на разделении процесса управления на два уровня:

\begin{itemize}
	\item \textbf{экономический уровень} — формирование справедливого распределения ресурсов посредством аукционного механизма;
	\item \textbf{адаптивный уровень} — корректировка стратегий агентов на основе обучения с подкреплением.
\end{itemize}

Таким образом, система рассматривается как иерархическая многоагентная структура:

\begin{equation}
	\mathcal{H} = \langle \mathcal{A}_{auction}, \mathcal{A}_{RL}, \mathcal{E} \rangle,
\end{equation}

где:
\begin{itemize}
	\item $\mathcal{A}_{auction}$ — аукционный слой;
	\item $\mathcal{A}_{RL}$ — обучающий слой;
	\item $\mathcal{E}$ — среда.
\end{itemize}

\section{Формализация гибридной модели}

\subsection{Расширенная Dec-POMDP модель}

Модель расширяется за счет включения экономического механизма:

\begin{equation}
	\text{Dec-POMDP}' = \langle N, S, A, P, R', O, \Omega, \gamma, h \rangle,
\end{equation}

где новая функция вознаграждения определяется как:

\begin{equation}
	R' = R_{RL} + \lambda \cdot R_{auction}.
\end{equation}

Здесь:
\begin{itemize}
	\item $R_{RL}$ — стандартное вознаграждение MARL;
    \item $R_{auction}$ — вклад аукционного механизма;
	\item $\lambda$ — коэффициент балансировки.
\end{itemize}

\subsection{Интеграция аукционного механизма}

Аукционный механизм формирует распределение ресурсов:

\begin{equation}
	x^* = \arg\max_x SW(x),
\end{equation}

где $SW(x)$ определяется в главе 2.

Результат аукциона влияет на:
\begin{itemize}
	\item допустимое пространство действий агентов;
	\item функцию вознаграждения;
	\item состояние среды.
\end{itemize}

Алгоритм функционирует в двух режимах: \textbf{режиме онлайн-исполнения} (Online Phase) и \textbf{режиме оффлайн-обучения} (Offline Phase).

\subsection{Режим 1: Онлайн-исполнение}

Онлайн-фаза работает в реальном времени, без доступа к глобальному состоянию системы. 
\begin{enumerate}
	\item \textbf{Формирование задач и наблюдений}. В момент времени $t$, каждое устройство $d_i$ формирует список задач $\tau$, а каждый edge-узел $n_j$ при этом формирует своё локальное наблюдение $o_j^t$.
	\item \textbf{Проведение аукциона}:
	\begin{itemize}
		\item сбор заявок $b_i(t)$ на выполнение задач;
		\item рассчет стомости $C_j(t)$ и полезности $U_i(\tau)$ выполнение задач;
		\item определение оптимального распределения $x^*$.
	\end{itemize}
	\item \textbf{Формирование вознаграждения с учетом VCG}: Для каждого устройства $d_i$  расчитываются VCG платежи $p_i$. Рассчитанные платежи интегрируются в функцию вознаграждения $r_j$ узлов. 
	\item \textbf{Назначение задач на узлы}. Каждый edge-узел принимает решение о взятии задачи на исполнение на основе своего локального наблюдения:
	\begin{enumerate}
		\item \textbf{Вычисление локальной Q-функции}. Каждый узел имеет свою нейросеть, которая вычисляет Q-значения $\mathbf{Q}_j(t)$ для всех возможных действий $a_j$.
		\item \textbf{Выбор действия и исполнение}. Узел выбирает действие на основе $\epsilon$-жадной стратегии, далее либо узел берет задачу на исполнение, либо нет.
	\end{enumerate} 
	\item \textbf{Накопление опыта}. Каждый переход сохраняется в Experience Reply Buffer: $o^t, a^t, r^t, o^{t+1}$.
\end{enumerate}

\subsection{Режим 2: Оффлайн-обучение}

Оффлай-фаза включает \textbf{обучение сетей QMIX}. На основе Experience Reply Buffer с обновлёнными вознаграждениями, система обучает сети QMIX путём минимизации TD-ошибки.

\subsection{Псевдокод алгоритма}

\begin{verbatim}
	Initialize Q-networks and replay buffer
	for each episode do
	observe state s
	for each agent i do
	select action a_i using epsilon-greedy
	end for
	
	run auction mechanism:
	create bids b_i(t)
	compute costs C_j(t) and utilities U_i(task)
	determine allocation x*
	
	execute actions and observe reward r
	store transition (s, a, r, s') in buffer
	
	sample minibatch from buffer
	update Q-networks using QMIX
	
	update target networks
	end for
\end{verbatim}

\subsection{Поток данных: пример со сценарием}

\textbf{Время $t=0$ (онлайн-фаза)}.

Устройство $d_1$ приходит с задачей $\tau_1$ (CPU=2, важность=0.8). Узлы имеют нагрузку 30\% и 20\% соответственно.

После проведения аукциона задача распределяется на узел. После вычисления Q-функций узлы выбирают действия. Задача $\tau_1$ назначается узлу с максимальным Q-значением для действия Accept. Система получает вознаграждение и сохраняет переход в буфер.

\textbf{Оффлайн-фаза (после 1000 переходов)}.

Система анализирует буфер и обнаруживает закономерности:
\begin{itemize}
	\item когда узел перегружен (load $> 0.4$), отклонение задач приводит к лучшему результату;
	\item платежи варьируются от 0.5 до 0.8 в зависимости от важности.
\end{itemize}

После чего система переобучает сети QMIX.

\section{Свойства предложенного метода}

\subsection{Справедливость распределения}

Использование VCG-механизма обеспечивает:
\begin{itemize}
	\item максимизацию социального благополучия;
	\item отсутствие стимулов к манипуляции;
	\item справедливое распределение ресурсов.
\end{itemize}

\subsection{Адаптивность}

Интеграция MARL позволяет:
\begin{itemize}
	\item адаптироваться к изменениям нагрузки;
	\item учитывать динамику сети;
	\item обучаться на основе опыта.
\end{itemize}

\subsection{Оптимальность}

Локальное решение узла согласовывается с глобально оптимальным распределением:
\begin{equation}
	\text{LocalDecision}(a_j) \approx \text{GlobalOptimal}(x^*)
\end{equation}

Это достигается через:
\begin{enumerate}
	\item \textbf{QMIX монотонность}: локальная оптимизация $\rightarrow$ глобальная оптимизация;
	\item \textbf{VCG платежи в вознаграждениях}: агенты мотивированы принимать глобально оптимальные решения;
	\item \textbf{Итеративное обучение}: параметры адаптируются к истинным функциям.
\end{enumerate}

\subsection{Масштабируемость}

Благодаря QMIX:
\begin{itemize}
	\item сложность составляет $O(k)$;
	\item обеспечивается работа с десятками и сотнями агентов.
\end{itemize}

\subsection{Устойчивость к неполной информации}

Использование Dec-POMDP обеспечивает:
\begin{itemize}
	\item корректную работу при локальных наблюдениях;
	\item устойчивость к шуму и задержкам.
\end{itemize}

\section{Анализ вычислительной сложности}

Сложность метода определяется как:

\begin{equation}
	O(k \cdot |A| + Auction(k)),
\end{equation}

где:
\begin{itemize}
	\item $|A|$ — размер пространства действий;
	\item $Auction(k)$ — сложность аукционного механизма.
\end{itemize}

В случае использования эффективной реализации аукциона:

\begin{equation}
	Auction(k) = O(k \log k).
\end{equation}

Таким образом, общий алгоритм сохраняет линейно-логарифмическую масштабируемость.

\section{Отличие от существующих подходов}

Предложенный метод отличается тем, что:

\begin{itemize}
	\item впервые объединяет VCG и QMIX в единой модели;
	\item учитывает как экономические, так и адаптивные аспекты;
	\item обеспечивает одновременно справедливость и адаптивность;
	\item применим в условиях частичной наблюдаемости.
\end{itemize}

\chapter{ПРАКТИЧЕСКОЕ ПРИМЕНЕНИЕ}\label{ch:ch6}

\section{Области применения}

Разработанный гибридный метод адаптивного управления ресурсами ориентирован на применение в распределенных системах, характеризующихся высокой динамичностью, гетерогенностью и ограниченной наблюдаемостью.

К основным областям применения относятся:

\begin{itemize}
	\item роботизированные складские системы;
	\item интеллектуальные транспортные системы;
	\item промышленные IoT-системы (Industry 4.0);
	\item системы умного города;
	\item распределенные вычислительные платформы edge/fog computing.
\end{itemize}

\section{Применение в роботизированных складских системах}

\subsection{Описание предметной области}

Современные складские комплексы активно используют автономных мобильных роботов (Autonomous Mobile Robots, AMR) (см. рис. \ref{img:example-amr})  для выполнения операций транспортировки, сортировки и обработки грузов.

\begin{figure}[h]
	\centering
	\includegraphics[width=14cm, height=8cm]{example-amr}
	\caption{Роботизированный склад}
	\label{img:example-amr}
\end{figure}

Каждый робот представляет собой интеллектуального агента, способного:
\begin{itemize}
	\item принимать решения в реальном времени;
	\item взаимодействовать с другими роботами;
	\item обрабатывать данные с сенсоров;
	\item выполнять вычислительные задачи.
\end{itemize}

\subsection{Постановка задачи}

В рамках складской системы возникает задача распределения вычислительных ресурсов между роботами для выполнения следующих задач:

\begin{itemize}
	\item обработка видеопотоков (распознавание объектов);
	\item планирование маршрутов;
	\item координация движения;
	\item предотвращение столкновений.
\end{itemize}

Данная задача характеризуется:
\begin{itemize}
	\item высокой динамичностью (роботы перемещаются);
	\item ограниченными ресурсами;
	\item жесткими требованиями к задержкам.
\end{itemize}

\subsection{Применение предложенного метода}

В рамках предложенного подхода:

\begin{itemize}
	\item роботы выступают как агенты устройств;
	\item edge-узлы — как вычислительные ресурсы;
	\item задачи распределяются через аукционный механизм;
	\item стратегии адаптируются с помощью MARL.
\end{itemize}

\subsection{Результаты применения}

Использование метода позволяет:

\begin{itemize}
	\item снизить задержки обработки данных;
	\item повысить устойчивость системы;
	\item обеспечить равномерное распределение нагрузки;
	\item уменьшить вероятность конфликтов между роботами.
\end{itemize}

\section{Применение в интеллектуальных транспортных системах}

\subsection{Описание системы}

Интеллектуальные транспортные системы (ITS) (см. рис. \ref{img:example-its}) включают множество IoT-устройств:

\begin{itemize}
	\item камеры видеонаблюдения;
	\item датчики трафика;
	\item светофоры;
	\item бортовые устройства автомобилей.
\end{itemize}

\begin{figure}[h]
	\centering
	\includegraphics[width=14cm, height=10cm]{example-its}
	\caption{Интеллектуальные транспортные системы (ITS)}
	\label{img:example-its}
\end{figure}

\subsection{Проблематика}

Основные задачи:

\begin{itemize}
	\item анализ дорожной обстановки;
	\item обнаружение аварий;
	\item прогнозирование пробок;
	\item адаптивное управление светофорами.
\end{itemize}

\subsection{Применение метода}

Предложенный метод позволяет:

\begin{itemize}
	\item распределять вычислительные задачи между edge-узлами;
	\item учитывать задержки передачи данных;
	\item адаптироваться к изменению транспортных потоков;
	\item обеспечивать справедливое распределение ресурсов.
\end{itemize}

\section{Применение в системах умного города}

\subsection{Сценарии использования}

В системах умного города метод может использоваться для:

\begin{itemize}
	\item управления энергопотреблением;
	\item мониторинга окружающей среды;
	\item обработки данных видеонаблюдения;
	\item управления коммунальной инфраструктурой.
\end{itemize}

\subsection{Преимущества метода}

\begin{itemize}
	\item децентрализация управления;
	\item масштабируемость;
	\item устойчивость к отказам;
	\item эффективное использование ресурсов.
\end{itemize}

\section{Применение в промышленности (Industry 4.0)}

\subsection{Особенности среды}

Промышленные IoT-системы характеризуются:

\begin{itemize}
	\item большим количеством датчиков;
	\item высокой частотой генерации данных;
	\item требованиями к надежности;
	\item строгими временными ограничениями.
\end{itemize}

\subsection{Роль метода}

Метод позволяет:

\begin{itemize}
	\item распределять задачи между периферийными узлами;
	\item минимизировать задержки;
	\item адаптироваться к изменениям производственной нагрузки;
	\item оптимизировать использование ресурсов.
\end{itemize}

\section{Оценка экономической эффективности}

Экономическая эффективность применения метода проявляется в:

\begin{itemize}
	\item снижении затрат на вычислительные ресурсы;
	\item уменьшении времени выполнения задач;
	\item повышении общей производительности системы;
	\item снижении количества отказов.
\end{itemize}

\section{Ограничения и перспективы применения}

Несмотря на преимущества, метод имеет ограничения:

\begin{itemize}
	\item необходимость обучения модели;
	\item вычислительные затраты на обучение;
	\item зависимость от параметров среды.
\end{itemize}

Перспективы дальнейшего развития:

\begin{itemize}
	\item интеграция с федеративным обучением;
	\item учет безопасности и доверия между агентами;
	\item применение в масштабных распределенных системах.
\end{itemize}
